🧱 v2 新增 3 大核心模块
1️⃣ Schema Auto Loader（自动建模）
🎯 目标

不用你手写 schema.json

系统自动从数据库获取：

表
字段
类型
注释
🧩 支持数据源
✔ MySQL
✔ PostgreSQL
✔ ClickHouse（你很适合这个）
✔ Hive（重点）
📦 输出结构
{
  tables: [
    {
      name: "user_table",
      columns: [
        { name: "user_id", type: "bigint" },
        { name: "create_time", type: "timestamp" }
      ]
    }
  ]
}
⚙️ 实现方式（简单版）
Step 1：连接 DB
SELECT table_name
FROM information_schema.tables;
Step 2：拉字段
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'xxx';
💡 v2关键点

schema 不再“写”，而是“读”

2️⃣ Semantic Layer（语义层 / 业务理解）

这是 v2 最大升级点。

🎯 目标

AI 不只是知道：

user_table
order_table

还要知道：

“业务是什么意思”

📦 语义层结构
{
  "metrics": [
    {
      "name": "新增用户",
      "logic": "user_table.create_time >= today - 7"
    },
    {
      "name": "GMV",
      "logic": "sum(order_table.pay_amount)"
    }
  ]
}
💡 v2关键能力
❌ v1

AI猜

✅ v2

用户说：新增用户
AI：

识别 metrics → 展开逻辑 → 生成 SQL
3️⃣ Query Orchestrator（查询编排器）

这是 v2 的“大脑”。

🎯 作用

负责：

判断问题类型
选择 schema / metrics
组装 prompt
控制多步推理
🧠 工作流程
用户问题
   ↓
Query Classifier
   ↓
判断类型：
  - 指标查询
  - 明细查询
  - join查询
   ↓
加载 context
   ↓
LLM生成SQL
🧩 示例
输入：

“最近7天新增用户和GMV”

Orchestrator行为：
识别：
- metrics query

加载：
- 新增用户定义
- GMV定义
- user + order schema

生成 prompt
🧠 v2 Prompt（升级版）
你是企业级数据分析SQL Agent。

你必须遵循：

=== Schema ===
{auto_schema}

=== Relations ===
{auto_relations}

=== Metrics ===
{semantic_metrics}

=== Query Type ===
{query_type}

规则：
1. 只允许使用schema中字段
2. 必须遵守关系
3. 优先使用metrics定义
4. 不确定返回无法生成SQL
5. 只输出SQL
